Python的Boolean探秘

Python里的Boolean值

首先，看一下官方定义的False值。

  • None
  • False
  • zero of any numeric type, for example, 0, 0L, 0.0, 0j.
  • any empty sequence, for example, ‘’, (), [].
  • any empty mapping, for example, {}.
  • instances of user-defined classes, if the class defines a __nonzero__() or __len__() method, when that method returns the integer zero or bool value False. [1]

与java，c++等相比，特别需要注意的是， python将 空序列，空字典，以及实现了 __nonzero__() or __len__()这两种方法的类（len实现实际也是返回0），都认为是 False.

考虑一下代码:

if []:
    print 'Not display'

if [1, 2, 3]:
    print 'will display'

if {}:
    print 'not display'

所以判断一个数组，字典是否为空，你可以写得pythnic一点，而不是len(somelist) > 0

Boolean 操作符

Operation Result
x or y if x is false, then y, else x
x and y if x is false, then x, else y
not x if x is false, then True, else False

这又是跟其他语言不一样的黑魔法。
考虑以下例子：

print 5 and 'a'  # 'a'
print '' or 'sb' # 'sb'

上面情况很容易跟Python三元操作弄混淆。

print '今天我们去大保健' if money.enough else '今天我们不去大保健'

当然，and or not, 我们最多的还是用来操作 Ture False。当然也是有截断操作的，Fasle or whatever.

比较操作符

Operation Meaning
< strictly less than
<= less than or equal
> strictly greater than
>= greater than or equal
== equal
!= not equal
is object identity
is not negated object identity

以前，不等于 != 还可以写作 <> 不过是为了保持以往代码的可用性，现在应该写成 !=

考虑以下代码：

a = [1,2,3]
b = [1,2,3]

a == b   # True
a is b   # False

c = 2
d = 2

c == d # True
c is d # True

判断两个数字，不应该使用 is 符号。这个部分比较trick，现在的水平解释不清楚，就不谈了。
